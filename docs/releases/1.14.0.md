## TPW Core v1.14.0 - Members Manager Permission

This release introduces a dedicated members-management permission, keeps the members admin flow aligned around a shared access helper, and hardens protected permission fields against privilege escalation.

### Added

- Added the core member field `is_manage_members` with the label `Members Manager`.

### Changed

- Access to the front-end members management UI and related capability checks now allows WordPress admins, TPW members with `is_admin = 1`, or TPW members with `is_manage_members = 1`.
- `is_manage_members` now follows the same Core checkbox-style handling pattern as other permission-style member flags across schema upgrade, field settings, profile protections, and add/edit forms.
- Protected permission fields `is_admin` and `is_manage_members` now remain visible but read-only for non-administrator managers, with server-side enforcement to block privilege escalation through form tampering or AJAX requests.

See `CHANGELOG.md` under [1.14.0] - 2026-03-10 for full details.
