## TPW Core v1.13.0 - Central Email Logging

This release adds a persistent, lightweight email logging system to TPW Core so administrators can inspect dispatcher activity, verify timing, and troubleshoot delivery failures from one generic core location.

### Added

- New `tpw_email_logs` table stores operational email log records with timestamp, recipient, subject, context, send status, error detail, and duration.
- New Email Logs tab under TPW Core Settings shows the latest 100 dispatcher log entries in newest-first order.
- New Clear Logs action lets administrators empty the log table directly from the Email Logs tab.
- Developer documentation now explains the dispatcher logging flow, the stored fields, optional context usage, and the retention policy.

### Changed

- `TPW_Email::dispatch_mail()` now records real send outcomes around `wp_mail()` and captures WordPress mail failure detail when available.
- Email log retention is enforced automatically through a daily cleanup that removes entries older than 30 days.

See `CHANGELOG.md` under [1.13.0] - 2026-03-10 for full details.
