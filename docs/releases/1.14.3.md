This release adds the first formal design specification for the upcoming TPW Core Sign-Up System. It gives site owners, implementers, and plugin teams a shared blueprint for how payment-first onboarding will work across TPW products, with a strong focus on recoverability, plugin extensibility, and safe post-payment account creation.

### Main changes

- Added a new Sign-Up System architecture specification to TPW Core documentation.
- Defined the generic Core lifecycle for signup attempts, including retry, recovery, expiry, locking, and finalization handling.
- Documented support for plugin-defined form sections, repeatable groups, validation rules, and finalization callbacks from day one.
- Clarified the payment-first onboarding model, where accounts are created only after successful payment and password setup happens afterwards.
- Outlined the planned Phase 1 operational UI for signup configuration and recovery tooling.

This release is primarily a documentation and architecture milestone, but it is an important step toward a consistent onboarding framework for FlexiSubscriptions and future TPW plugins.
